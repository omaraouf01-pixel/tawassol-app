## 🎯 البرومت الرئيسي (انسخه كاملاً)

```
أنا مطوّر جديد على مشروع Tawassol (Next.js + Firebase) وعندي 4 أيام فقط لأفهم
كامل الكود: كل صفحة، كل مكوّن، كل دالة، كل زر، وكل عملية (تسجيل، إضافة عضو،
إنشاء مجموعة، الدردشة، الموافقة من الأدمن… إلخ).

مهمتك: اشرح لي التطبيق بالكامل بأسلوب منظّم، واكتب الشرح في ملف اسمه
`APP_GUIDE.md` في جذر المشروع. اتبع الخطة التالية بالضبط (يوم بيوم)، ولا تنتقل
لليوم التالي إلا بعد أن أؤكد لك.

═══════════════════════════════════════════════════════════
📅 اليوم 1 — الأساسيات والبنية العامة
═══════════════════════════════════════════════════════════
1. اقرأ: package.json, next.config.js, CLAUDE.md, PROJECT_MAP.md (إن وجد).
2. اكتب في APP_GUIDE.md قسم "نظرة عامة" يشمل:
   - الـ Stack المستخدم وكل مكتبة ودورها.
   - شجرة المجلدات مع شرح دور كل مجلد.
   - ملفات lib/ واحداً واحداً: اسم الملف + ماذا يصدّر + متى يُستخدم.
   - ملفات الإعداد (firebase config, cloudinary, tailwind).
3. ارسم خريطة التوجيه (Routing Map): كل مسار في app/ → ما هي الصفحة
   ومن يصل إليها (guest/pending/active/admin).

═══════════════════════════════════════════════════════════
📅 اليوم 2 — المصادقة والمستخدمون والصلاحيات
═══════════════════════════════════════════════════════════
لكل عملية من العمليات التالية اكتب في APP_GUIDE.md قسماً مستقلاً يحتوي:
  • اسم العملية
  • الملف/الملفات المسؤولة (مع المسار الكامل ورقم السطر إن أمكن)
  • الدوال المستخدمة (اسم الدالة + ماذا تفعل سطر بسطر مختصراً)
  • الأزرار/الـ UI Triggers التي تشغّلها
  • التدفق الكامل خطوة بخطوة (من ضغط الزر حتى تحديث Firestore)
  • الـ Collections في Firestore التي تتأثر

العمليات المطلوبة لليوم 2:
  1. تسجيل حساب جديد (signup)
  2. تسجيل الدخول (login) + معالجة الأخطاء (mapAuthError)
  3. تسجيل الخروج (logout)
  4. تدفق pending → موافقة الأدمن → active
  5. الـ onboarding وحقل onboarded
  6. useAuth() — كيف يقرّر التوجيه التلقائي
  7. حماية الصفحات (من يصل لماذا)

═══════════════════════════════════════════════════════════
📅 اليوم 3 — المجموعات والدردشة (Hub + Chat)
═══════════════════════════════════════════════════════════
نفس التنسيق (عملية / ملفات / دوال / أزرار / تدفق / collections) للعمليات:
  1. عرض قائمة المجموعات في Hub
  2. استكشاف المجموعات (Explore + JoinNodeModal)
  3. الانضمام لمجموعة
  4. فتح صفحة دردشة مجموعة + فحص الصلاحية (member/leader/admin)
  5. إرسال رسالة (نص + ملف عبر /api/upload + Cloudinary)
  6. عرض الأعضاء (MemberListDrawer, GroupMembers)
  7. لوحة الإشراف (ModerationPanel, OverseerPanel)
  8. إعدادات المجموعة (GroupSettings) — تعديل، إضافة/طرد عضو
  9. الموارد (ResourcesSidebar) والـ ActiveNodesSidebar

═══════════════════════════════════════════════════════════
📅 اليوم 4 — لوحة الأدمن + APIs + الخاتمة
═══════════════════════════════════════════════════════════
1. لوحة الأدمن: AdminUsersTable, AdminPendingTable, AdminGroupsTable
   (لكل جدول: الأعمدة، الأزرار، الدوال خلف كل زر، الـ API endpoint المستدعى).
2. كل ملف داخل app/api/** : المسار + الـ method + ماذا يفعل + من يستدعيه.
3. apiClient.js: كيف يُلصق التوكن، كيف تُعالج الأخطاء.
4. الإعدادات (SettingsMenu) والشريط الجانبي (Sidebar).
5. في نهاية APP_GUIDE.md أضف:
   • "فهرس الدوال" — جدول بكل دالة مهمة (الاسم | الملف | السطر | ماذا تفعل).
   • "فهرس الأزرار" — جدول بكل زر رئيسي (الزر | الصفحة | الدالة المرتبطة | النتيجة).
   • "خريطة Firestore Collections" — كل collection + الحقول + من يكتب/يقرأ.
   • "المخاطر والـ Gotchas" (race conditions, re-renders, إلخ).
```
